# Plan: Fix white screen on Android 7 WebView

## Problem
Приложение показывает белый экран на Android 7 в WebView нативного приложения. Android 7 (Nougat) использует системный WebView на базе **Chrome 51**.

## Root Cause Analysis

### 1. `@vitejs/plugin-legacy` установлен, но НЕ настроен
`@vitejs/plugin-legacy` есть в `devDependencies` корневого `package.json`, но **не используется** ни в одном `vite.config.ts`. Без него:
- Нет SystemJS-фолбэка для браузеров без нативной поддержки ES-модулей
- Нет автоматической инъекции полифиллов через `core-js`
- Chrome 51 **не поддерживает** нативные ES-модули (`<script type="module">` появился в Chrome 61)

### 2. Полифиллы подключены, но не импортированы
`core-js` и `regenerator-runtime` есть в `dependencies` guest app, но **нигде не импортируются** в `main.tsx`. Они бесполезны без явного `import 'core-js/stable'`.

### 3. Chrome 51 не поддерживает современные API
Даже с `build.target: 'es2015'` (синтаксис транспилируется esbuild), Chrome 51 не имеет ряда современных JS/CSS API.

## Implementation Plan

### Step 1: Configure `@vitejs/plugin-legacy` in `apps/guest/vite.config.ts`

**File**: `apps/guest/vite.config.ts`

Добавить legacy-плагин для генерации SystemJS-фолбэк-бандлов:

```ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import legacy from '@vitejs/plugin-legacy'
import path from 'path'

export default defineConfig({
  plugins: [
    react(),
    legacy({
      targets: ['Chrome >= 51'],
      additionalLegacyPolyfills: ['regenerator-runtime/runtime'],
    }),
  ],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
  build: {
    target: 'es2015',
  },
  server: {
    port: 5173,
  },
})
```

### Step 2: Add polyfill import to `apps/guest/src/main.tsx`

**File**: `apps/guest/src/main.tsx`

Добавить импорт `core-js` в самом начале (до всех остальных импортов):

```tsx
import 'core-js/stable'
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App'
import './index.css'

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
)
```

### Step 3: Apply same changes to admin app (optional, for consistency)

**File**: `apps/admin/vite.config.ts` — same legacy plugin config
**File**: `apps/admin/src/main.tsx` — same core-js import (if admin is also served on Android)
